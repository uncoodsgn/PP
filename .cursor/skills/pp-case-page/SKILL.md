# Страница кейса PP

## Шаги

1. Открой любой готовый файл `case-*.html` (например `case-pillar.html`) как эталон разметки шапки, `main`, подключения `style.css` и шрифтов.
2. Скопируй структуру: `<!DOCTYPE html>`, `lang="ru"`, те же `preconnect` и `link` на шрифты и `style.css`, класс `interfaces-page` на `body` (для раздела Graphics & 3D добавь ещё `graphics-3d-page`), инлайн-скрипт курсора при необходимости.
3. В `work-header`: ссылка «Back» (`history.back()`), заголовок на `index.html`, «Next Case» — на соседний логичный `case-*.html` (согласуй с пользователем цепочку кейсов).
4. Заголовок кейса в `work-case-title`, подзаголовок со ссылкой на Behance/внешнюю страницу при наличии — как в соседних кейсах.
5. Медиа: положи файлы в `img/`, в галерее используй классы `zin-gallery-wrap`, `zin-gallery`, `zin-gallery-slide` (количество слайдов — по аналогии с похожим кейсом: `--6`, зин и т.д.).
6. Проверь относительные пути и что новая страница связана с `interfaces.html` / `graphics-3d.html` / списками, если это требуется.

## Не делать

- Не подключать фреймворки ради одной страницы.
- Не менять глобально `style.css` ради одного кейса без явной необходимости — сначала модификаторы/локальные обёртки.
